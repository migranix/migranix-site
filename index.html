Four Ways to Move Your Data

From one-time migrations to real-time CDC — pick the right mode for your use case.

1

Full Load

Extract all rows, truncate target, reload. Best for initial migration or small tables. Batched for large datasets.

2

Incremental

Sync only changed rows using a watermark column like updated_at. Auto-detects best column and primary keys.

3

CDC — SQL Server

Read SQL Server transaction logs via fn_cdc_get_all_changes. Tracks LSN position. Captures INSERT, UPDATE, and DELETE.

4

CDC — PostgreSQL

Logical replication using pg_logical_slot_get_changes and wal2json. Creates publication and slot automatically.
